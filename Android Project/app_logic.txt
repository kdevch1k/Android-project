Budget Buddy - Логика приложения

1. Основные классы

MainActivity:
- Отображение списка транзакций через RecyclerView
- Подсчет и отображение текущего баланса
- Обработка нажатия FAB для быстрого добавления
- Обновление данных при возврате из Quick/AddTransactionActivity
- Фильтрация транзакций (базовая версия - только по типу)
- Управление анимациями и переходами

QuickInputActivity:
- Быстрый ввод транзакции
- Управление цифровой клавиатурой
- Автоматическое применение последних настроек
- Вибрация при нажатии кнопок
- Анимации переходов

AddTransactionActivity:
- Полный ввод транзакции
- Валидация введенных данных
- Сохранение транзакции в БД
- Обработка выбора категории
- Форматирование и парсинг суммы
- Обработка выбора даты

TransactionAdapter:
- Отображение списка транзакций
- Форматирование дат и сумм
- Группировка по датам
- Обработка свайпов для удаления
- Анимации при обновлении списка

2. Модели данных

Transaction:
- id: Long
- type: TransactionType (INCOME, EXPENSE)
- amount: BigDecimal
- date: Date
- categoryId: Long
- note: String
- createdAt: Date
- lastModified: Date

Category:
- id: Long
- name: String
- type: TransactionType
- iconId: Int
- isDefault: Boolean
- usageCount: Int
- lastUsed: Date

3. База данных

AppDatabase:
- Singleton для доступа к БД
- Методы миграции
- Предзаполнение базовыми категориями
- Шифрование данных

TransactionDao:
- Добавление транзакции
- Получение списка транзакций
- Подсчет баланса
- Группировка по категориям
- Получение последних транзакций

CategoryDao:
- Получение категорий по типу
- Добавление/редактирование категорий
- Обновление статистики использования
- Получение топ-5 категорий

4. Утилиты

DateUtils:
- Форматирование дат
- Получение начала/конца периода
- Группировка по периодам
- Проверка корректности дат

CurrencyUtils:
- Форматирование сумм
- Парсинг введенных значений
- Валидация сумм
- Работа с калькулятором

SecurityUtils:
- Шифрование базы данных
- Работа с Android Keystore
- Защита от скриншотов

5. Валидация данных

Проверки при добавлении транзакции:
- Сумма больше нуля
- Категория выбрана
- Дата не в будущем
- Заметка не слишком длинная

6. Обработка ошибок

Основные сценарии:
- Ошибка сохранения в БД
- Некорректный ввод суммы
- Отсутствие категорий
- Ошибка загрузки данных

7. Оптимизация производительности:
- Кэширование часто используемых данных
- Эффективная работа с базой данных
- Отложенная загрузка старых транзакций
- Оптимизация анимаций
